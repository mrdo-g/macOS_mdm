# Расширенный обход MDM для macOS

**Автор:** Martin Grigoryan  
**Контакты:** Instagram [@_mrdo_g](https://instagram.com/_mrdo_g) • Telegram @mrd0_g  
**Версия скрипта:** 1.3 (MDM Bypass)

---

## Описание

Этот скрипт предназначен для **полного обхода профилей управления мобильными устройствами (MDM)** на компьютерах Mac.  
Работает из режима восстановления (Recovery mode) и выполняет комплекс мер по подавлению MDM-клиента, блокировке серверов Apple и очистке системных следов.

---

## Технические возможности

| Функция | Что делает |
|---------|-------------|
| **Обнаружение томов** | Автоматически находит системный и data-тома macOS (без переименования). |
| **Разблокировка FileVault** | Запрашивает пароль для расшифровки data-тома, если включено шифрование. |
| **Создание локального администратора** | Позволяет задать имя, логин и пароль (по умолчанию: `Apple` / `Apple` / `1234`). |
| **Блокировка MDM-доменов** | Добавляет в `/etc/hosts` 5 доменов Apple (IPv4 и IPv6): `deviceenrollment.apple.com`, `mdmenrollment.apple.com`, `iprofiles.apple.com`, `acmdm.apple.com`, `axm-adm-mdm.apple.com`. |
| **Очистка MDM-маркеров** | Удаляет положительные флаги активации, создаёт и защищает негативные маркеры, правит `com.apple.ManagedClient.plist`, удаляет `/var/db/mdm`. |
| **Принудительное отключение демона** | Создаёт файл `/var/db/.com.apple.mdmclient.daemon.forced_disable` с правами `000` и флагом `uchg`. |
| **Отключение MDM-служб** | Использует `launchctl disable` и `bootout` для всех системных и пользовательских агентов MDM. |
| **Удаление сторонних MDM‑агентов (опционально)** | Сканирует LaunchDaemons/Agents на наличие слов: `addigy`, `jamf`, `kandji`, `mosyle`, `intune`, `falcon`, `dorthus`, `jumpcloud`. |
| **Сброс сетевых настроек (опционально)** | Удаляет сохранённые Wi-Fi пароли, VPN и другие сетевые конфигурации. |
| **Управление root (опционально)** | Может сменить пароль root или полностью отключить учётную запись. |

## Требования

* **macOS** (Intel или Apple Silicon).
* **Режим восстановления** (Recovery mode) – настоятельно рекомендуется. При запуске из обычной системы выводится предупреждение.
* **Права root** – скрипт проверяет `EUID` и завершится с ошибкой, если запущен не от root.
* **Подключение к интернету** (через Wi-Fi в Recovery) – необходимо, если вы планируете скачать скрипт через `curl`.

---

## Инструкция по использованию

### 1. Загрузитесь в режим восстановления

* **Apple Silicon (M1/M2/M3/M4/M5):** выключите Mac → зажмите кнопку питания → выберите «Параметры» → нажмите «Продолжить».
* **Intel:** выключите Mac → включите и сразу зажмите **Command + R** до появления логотипа Apple.

### 2. Подключитесь к Wi-Fi

В правом верхнем углу экрана восстановления выберите нужную сеть Wi‑Fi и введите пароль.

### 3. Откройте Терминал

В панели меню выберите **«Утилиты» (Utilities) → «Терминал» (Terminal)**.

### 4. Загрузите и запустите скрипт

**Вариант А – через curl (если репозиторий приватный, сначала сделайте скрипт доступным по прямой ссылке):**

```bash
curl -L -o macos_mdm.sh https://github.com/mrdo-g/macOS_mdm/macos_mdm.sh && chmod +x macos_mdm.sh && sudo ./macos_mdm.sh
```

**Вариант Б – с флешки:**

1. На другом компьютере скачайте файл `macos_mdm.sh` и скопируйте его на USB-накопитель.
2. Вставьте флешку в Mac, находящийся в режиме восстановления.
3. В Терминале выполните:
   ```bash
   cp /Volumes/ИМЯ_ФЛЕШКИ/macos_mdm.sh /tmp/ && cd /tmp && chmod +x macos_mdm.sh && sudo ./macos_mdm.sh
   ```
   (Замените `ИМЯ_ФЛЕШКИ` на реальное имя вашего USB-накопителя.)

### 5. Что происходит во время выполнения скрипта

Скрипт выполняет шаги **автоматически**, только некоторые действия требуют вашего ответа:

1. Проверяет окружение (предупреждает, если вы не в Recovery).
2. Находит системный и data-тома (если найдено несколько – предложит выбрать).
3. **При необходимости разблокирует FileVault** – попросит пароль.
4. **Создаёт локального администратора:**
   * Запрашивает `Полное имя` (Enter = `Apple`)
   * Запрашивает `Имя учётной записи` (Enter = `Apple`)
   * Запрашивает `Пароль` (Enter = `1234`)
5. Блокирует MDM-домены в `/etc/hosts`.
6. Очищает и блокирует MDM-маркеры.
7. Отключает MDM-службы через `launchctl`.
8. **Опциональные вопросы** (отвечайте `y` или `n`):
   * «Обнаружены следы сторонних MDM… Удалить их?»
   * «Очистка сетевых конфигураций удалит сохранённые Wi-Fi пароли… Выполнить очистку?»
   * «Сменить пароль root?»
   * «Отключить учётную запись root?»
9. Выводит имя и пароль созданного пользователя.
10. Спрашивает: «Перезагрузить сейчас?» (по умолчанию `N`).

## После перезагрузки

1. **Войдите** под созданным администратором (например, `Apple` с паролем `1234`).
2. Пропустите все шаги первоначальной настройки (Apple ID, Siri, аналитика, экранное время и т.д.).
3. **Создайте свою основную учётную запись администратора**:
   * Откройте «Системные настройки» → «Пользователи и группы».
   * Нажмите на замок, введите пароль текущего пользователя (`Apple`).
   * Нажмите «+» → выберите «Администратор», заполните поля.
4. **Выйдите из временного пользователя** (меню Apple → «Выход…») и зайдите под новым.
5. **Удалите временного пользователя** (при желании) через «Системные настройки» → «Пользователи и группы».
6. Готово! MDM больше не должен беспокоить.

> ⚠️ При последующих обновлениях macOS **не удаляйте** созданные скриптом защитные файлы в `/var/db/ConfigurationProfiles/Settings/` и строки в `/etc/hosts`. Они сохраняют блокировку.

## Примечания и гарантии

* Скрипт не меняет серийный номер или аппаратные параметры – это не является нарушением MIC, а лишь локально подавляет корпоративное управление.
* На некоторых конфигурациях может потребоваться **отключение SIP** (System Integrity Protection) для полного контроля над `/etc/hosts` и системными демонами. Если скрипт не может заблокировать файлы, он выдаст предупреждение, но продолжит работу.
* Для повторного включения MDM (например, перед продажей компьютера) необходимо вручную удалить все защитные файлы, созданные скриптом, и снять блокировки. Инструкция по обратному процессу не предоставляется.
* Автор не несёт ответственности за неправильное использование скрипта или нарушение корпоративной политики. Используйте на свой страх и риск.
